In Buildkit, a "build type" defines the download and installation process for a complete, working, testable system. For example, buildkit includes a few pre-defined build types:

  • drupal-clean: A basic, minimalist installation of Drupal 7 and CiviCRM
  • drupal-demo: The standard Drupal-CiviCRM demo used for drupal.demo.civicrm.org which includes a special home page, theme configuration, the CiviVolunteer extension, etc.
  • wp-demo: The standard WordPress-CiviCRM demo used for wordpress.demo.civicrm.org which includes some example WP content, the CiviVolunteer extension, etc.

A build-type is basically a pair of two shell scripts (e.g. app/config/drupal-clean/download.sh and app/config/drupal-clean/install.sh).

download.sh

The download instructions can be written many different ways, but these are a few common approaches:
  • Assemble build via bash: Download our preferred libraries and modules using bash commands like git clone, drush dl, wp plugin install, or even wget.
  • Assemble build via drush make: Download our preferred libraries and modules using drush make's special purpose language.
  • Big Old Git Repository of Everything (BOGRE): Manually copy all code (CMS code, CiviCRM code, modules, themes, etc) into a single git repository (eg {{cfg.bogre.url}}). Buildkit downloads that repository (and nothing else).
BOGRE may be useful for production sites where you only import production-ready code, but if you wish to actively participate in the development community, then BOGRE can be tiresome.
