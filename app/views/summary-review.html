Code Review Test is another variation on continuous integration which makes sense when your project requires review of any code changes. This scenario involves a sequence of steps:

  1. A developer prepares a change to the codebase (eg to the civicrm-core code or to one of your own extensions/modules/themes).
  2. The developer submits the change for code-review by opening a pull-request on github.com.
  3. github.com notifies your Jenkins server of a pending change.
  4. Jenkins downloads the application (and the proposed change), executes the tests, and posts the results back to github.com.
  5. The original developer (or any other developers) can see the test results on github.com and make a decision about whether to merge the results.
