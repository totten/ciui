drush make only works with Drupal.

CiviCRM UpgradeTest only works with Drupal.

Drupal SimpleTest only works with Drupal.

Basic Build downloads and installs CiviCRM along with a CMS. You can hack/code/test on a basic build.

Scheduled Test is useful for continuous-integration. For example, you might configure a Jenkins job to periodically download CiviCRM and execute a test-suite.

Code Review Test is useful for continuous-integration. For example, if a developer submits a pull-request on github.com, then github.com might notify your Jenkins server. Jenkins will then download CiviCRM (with patches), execute a test suite, and post the results on github.com.

In build type {{cfg.buildkit.type}}, the modules and libraries have been pre-selected by the fine folks at civicrm.org. For more options, see the configuration menu.
To pick modules and libraries for {{buildkitType()}}, we need to write a bash script with commands like git clone, drush dl, or wp plugin install.
To pick modules and libraries for {{buildkitType()}}, we need to write a drush make file.
Big Old Git Repository of Everything: Some developers prefer to manually copy all code (CMS code, CiviCRM code, modules, themes, etc) into a single git repository (eg {{cfg.bogre.url}}). In BOGRE mode, buildkit will download that repository.
