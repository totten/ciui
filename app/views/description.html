drush make only works with Drupal.

CiviCRM UpgradeTest only works with Drupal.

Drupal SimpleTest only works with Drupal.

Basic Build downloads and installs CiviCRM along with a CMS. You can hack/code/test on a basic build.

Panoramic Build downloads and installs multiple versions of CiviCRM (using the standard demo configurations). This is useful if you frequently work with different versions of Civi.

Scheduled Test is useful for continuous-integration. For example, you might configure a Jenkins job to periodically download CiviCRM and execute the test-suite.

Code Review Test is useful for continuous-integration. For example, if a developer submits changes for code-review (e.g. by opening a pull-request on github.com), then github.com might notify your Jenkins server. Jenkins will then download CiviCRM (with patches), execute the test suite, and post the results on github.com.

In build type {{cfg.buildkit.type}}, the modules and libraries have been pre-selected by the fine folks at civicrm.org. For more options, see the configuration menu.
For build type {{buildkitType()}}, we must automatically download our preferred libraries and modules. To do this, we will create a bash script with commands like git clone, drush dl, or wp plugin install.
For build type {{buildkitType()}}, we must automatically download our preferred libraries and modules. To do this, we will create a drush make file.
Big Old Git Repository of Everything: Some developers prefer to manually copy all code (CMS code, CiviCRM code, modules, themes, etc) into a single git repository (eg {{cfg.bogre.url}}). In BOGRE mode, buildkit will download that repository.
BOGRE may be useful for production sites where you only import production-ready code, but if you wish to actively participate in the development community, then BOGRE can be tiresome.
